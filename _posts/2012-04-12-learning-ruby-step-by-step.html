date = Date.today

2. 语法学习Part-1

  1. include, require, load的区别

    require和load用于加载文件，include用于加载模块。

    load加载的文件需要带后缀.rb，require不用。require多用于加载库文件，load多用于加载配置文件。

    include用于mix-in模块，减少书写长度。

    require 'webrick'
include 'webrick'

server = HTTPServer.new(...)
//可以不用 server = WEBrick::HTTPServer.new(...)

  2. Module的用法

    1. Namespaces

    提供了命名空间的作用来减少命名冲突。

    //trig.rb
module Trig
	PI = 3.14
	def Trig.sin (x)
		#...
	end
end
//action.rb
module Action
	BAD = 1
	def Action.sin (badness)
		#...
	end
end
//test.rb
require 'trig'
require 'action'

y = Trig.sin(Trig::PI/4)
wrongdoing = Action.sin(Action::BAD)

    2. Mixins

    通过Module来实现Mixins，作为多重继承的一种替代。

    module Debug
	def whoamI?
		"#{self.type.name} (\##{self.id}): #{self.to_s}"
	end
end
class Photograph
	include Debug
	#...
end
class EightTrack
	include Debug
	#...
end
ph = Photograph.new('West End Blues')
et = EightTrack.new('Pillow')
ph.whoamI?
et.whoamI?

  3. Class与OO

    类机制与面向对象等。

    class Song
	def initialize(name)
		@name = name
	end
end
song = Song.new("Rush")

    initialize是默认的初始化方法，唯一默认的私有方法。通过Song.new生成实例。@name为实例变量。

    变量类型和方法类型：

    @name - 实例变量

    @@name - 类变量

    def to_s - 实例方法

    def Song.to_s - 类方法

    继承

    class Jazz < Song
	def initialize(name, author)
		super(name)
		@author = author
	end
end
jazz = Jazz.new("Rush","Tom")

    Song为Jazz的父类。

    super关键字：带参数的super指调用父类实例变量或方法，不带参数的话，将调用父类的同名方法，传递给当前类方法的参数会默认的传给父类。

    访问控制

    public: 公共

    private: 私有

    protected: 受保护的，仅可以被定义的类或子类调用

    属性声明

    attr_reader: 对应get

    attr_writer: 对应set

    attr_accessor: 既可读也可写
